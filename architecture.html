Serviços e Responsabilidades

Frontend

payment-frontend

  • React 18 + Vite + TypeScript
  • HTTPS com certificado autoassinado (SAN)
  • SSE streaming chat
  • Authorization Code Flow (sem PKCE — HTTP LAN)
  • Token proxy via nginx (/kc/token) — sem mixed-content
  • PinPad — botão inline no chat, senha mascarada
  • Upload de imagens (galeria ou câmera) com resize canvas
  • Gravação de áudio push-to-hold → WAV → STT
  • Leitura de QR Code Pix (jsQR) via câmera ou galeria
  • Pix Tracker — stepper em tempo real (PENDING→SUCCESS)
  • Comprovante PDF download (BACEN-compliant)
  • Tabs: Chat, Transações, RAG, Voz, Agente
  • nginx proxy + Docker DNS resolver
Auth

Keycloak

  • Realm: payment
  • Client: payment-frontend (public)
  • Authorization Code Flow
  • redirectUris: localhost + 192.168.15.50 (HTTP e HTTPS)
  • webOrigins: ["+"] (espelha redirectUris)
  • Claims: name, given_name, family_name, email
  • Registro de novos usuários habilitado
Agente

payment-agent

  • Spring AI ChatClient + GPT-4o
  • MessageChatMemoryAdvisor (memória por sessão)
  • QuestionAnswerAdvisor (RAG)
  • 6 tools: openAccount, getBalance, getStatement, payPix, payBoleto, decodeQrCode
  • Multimodal: leitura de imagens (recibos, notas, QR)
  • TTS: POST /api/tts/speech → gpt-4o-mini-tts
  • STT: POST /api/stt/transcribe → gpt-4o-audio-preview (WAV)
  • Resolve conta por keycloakUserId ou email (sem perguntar)
  • accountNumber injetado via ToolContext
  • JWT validation via JWK Set URI (issuer agnóstico)
  • CORS: HTTP+HTTPS para localhost e RFC-1918
  • AgentPromptService: versionamento de prompts
Banking

payment-banking-service

  • AccountService: abertura, saldo, extrato
  • PaymentController: PIX por chave, boleto, QR Code
  • POST /api/payments/pix/qrcode/decode → EMV decoder
  • CelcoinClient: OAuth2 token cache automático
  • BCrypt: senha de transação (6 dígitos)
  • TransactionEventLogger: rastreamento de eventos Pix
  • PdfReceiptService: comprovante BACEN (OpenPDF)
  • PdfStatementStore: tokens de download (10 min)
  • Webhook handler: PAID/FAILED → atualiza tx + eventos
  • CORS: HTTP+HTTPS para localhost e RFC-1918
Mock

celcoin-mock

  • Banco próprio isolado (postgres-celcoin)
  • Onboarding: CBE001–CBE022 (CPF, idade, endereço)
  • DICT: GET /celcoin/pix/v2/dict/key/{key}
  • 7 chaves PIX seedadas (CPF, phone, email, EVP)
  • PIX transfer: PIE001–PIE010 + webhook assíncrono
  • Boleto: BLE001–BLE004
  • EMV Decoder: POST /pix/v1/emv/decode (estático e dinâmico)
  • OAuth2 mock: POST /v5/token
RAG

payment-rag

  • Upload de documentos (PDF, TXT, DOCX)
  • Chunking + embeddings via OpenAI
  • ChromaDB: coleção PaymentKnowledge
  • Versionamento de documentos por hash SHA-256
